# v1.3.2-Q1-motif-screening-status

## Scope

v1.3.2 adds a controlled Q1_Zn motif screening workflow.

Q1 initial generation already works. v1.3.1 diagnosed that a single selected Q1_Zn motif can lose the fourth Zn-O neighbor after static minimization. v1.3.2 therefore screens multiple topology-valid Q1 candidates instead of forcing the original candidate to pass.

## New Command

```text
python examples/10_screen_q1_motifs.py
```

Optional environment variables:

```text
PYCSH_ZN_Q1_SCREEN_TOP_N=6
PYCSH_ZN_Q1_REPOSITION_ZN=1
```

The reposition option is disabled by default. When enabled, repositioned variants are recorded separately and only move Zn toward the centroid of the four intended motif O atoms. The C-S-H framework, force-field parameters, and Zn-O bond topology are not changed.

## Outputs

```text
output_Y/workflow_v1/q1_motif_screening/q1_motif_screen_summary.csv
output_Y/workflow_v1/q1_motif_screening/q1_motif_screen_summary.json
output_Y/workflow_v1/q1_motif_screening/best_candidate.json
output_Y/workflow_v1/q1_motif_screening/screening_manifest.json
output_Y/workflow_v1/q1_motif_screening/candidates/candidate_XXXX/
```

Each candidate directory contains the candidate LAMMPS data file, Zn/water summaries, initial and post-min validation JSON, LAMMPS logs, CS-Info-reattached post-min data, pre/post Zn-O coordination comparison, and postprocess structure analysis when available.

## Ranking

Candidates are ranked by:

1. post-minimized `valid_q1_zn_candidate` first;
2. whether the four intended motif O atoms remain the closest four Zn-O neighbors;
3. smaller post-min max Zn-O distance;
4. smaller pre-min tetrahedral angle deviation;
5. lower clash score.

If no candidate passes, `best_candidate.json` still records the least-bad candidate and explicitly states that no screened Q1_Zn motif passed post-min validation.

## Current Smoke Test

A top-2 smoke test completed:

```text
PYCSH_ZN_Q1_SCREEN_TOP_N=2 python examples/10_screen_q1_motifs.py
```

Result:

```text
any_postmin_valid: true
best candidate: candidate_0002_base
postmin_classification: valid_q1_zn_candidate
post_coordination_2p5: 4
```

The first screened candidate still failed as `needs_static_relaxation`, showing why multi-candidate screening is necessary.

## Boundaries

- CementFF4-Zn force-field parameters are unchanged.
- The 2.5 A Zn-O coordination validation gate is unchanged.
- Validation semantics are unchanged.
- No finite-temperature MD is introduced.
- No MD-ready classification is introduced.
- mixed Q1+Q2b is not introduced.
- Q1 is not added to default mechanics.

## Mechanics Policy

Q1_Zn may only be considered for an opt-in mechanics workflow after at least one post-minimized screened Q1 structure validates as `valid_q1_zn_candidate`.
