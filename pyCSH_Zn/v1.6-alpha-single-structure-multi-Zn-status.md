# v1.6-alpha Single-Structure Multi-Zn Status

## Scope

v1.6-alpha introduces a minimal single-structure multi-Zn generator:

```text
examples/15_generate_multi_zn_structure.py
```

Supported alpha modes:

- `multi_q2b`: two independent Q2b_Zn motifs in one C-S-H structure.
- `multi_q1`: two independent Q1_Zn motifs in one C-S-H structure.
- `q1_q2b_single_structure_mixture`: one Q1_Zn motif and one Q2b_Zn motif in the same C-S-H structure.

This is single-structure multi-Zn / mixed-motif generation. It is not the old `mixed_Q1_Q2b_Zn` site type, which remains unsupported.

## Selection Constraints

Selected Zn sites must satisfy:

- no duplicate substituted Si site;
- PBC minimum-image Zn-Zn distance at least the requested `--min-zn-zn-distance`;
- no reuse of the same hydroxylated O core-shell pair;
- current Q1_Zn and Q2b_Zn topology checks;
- explicit CementFF4-Zn charge assignment and charge balance;
- CS-Info, water topology, and Zn-O coordination validation.

Failed candidate combinations are rejected with reasons. The script must not fall back to a single-Zn structure and report it as a multi-Zn success.

## Validation Labels

The validator now supports multi-Zn classifications without changing existing single-Zn labels:

- `valid_multi_q1_zn_candidate`
- `valid_multi_q2b_zn_candidate`
- `valid_multi_q1_q2b_zn_candidate`
- `failed_multi_zn_candidate`

Every Zn center must have at least four O neighbors within 2.5 Angstrom for a valid multi-Zn classification.

## Outputs

Default output directory:

```text
output_Y/workflow_v1/multi_zn_structure/
```

Main outputs:

- `multi_zn_cementff_zn.data`
- `multi_zinc_summary.json`
- `water_summary.json`
- `cementff_mapping_summary.json`
- `generation_manifest.json`
- `validation_initial.json`
- `validation_postmin.json` when `--run-static-relaxation` is used
- `multi_zn_pre_post_coordination_compare.json` when post-min validation is run
- `postprocess/structure_analysis.json` when postprocess succeeds

## Non-Scope

v1.6-alpha does not modify CementFF4-Zn force-field parameters, does not relax the 2.5 Angstrom Zn-O coordination threshold, does not introduce finite-temperature MD, does not introduce `md_ready_candidate`, does not claim final elastic constants, and does not claim production mechanical properties.

Q1_Zn is still not added to the default `examples/07_run_quasistatic_mechanics.py` targets.
