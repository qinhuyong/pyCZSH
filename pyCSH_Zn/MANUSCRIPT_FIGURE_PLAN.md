# Manuscript Figure Plan

This plan sketches manuscript figures for the pyCSH-Zn workflow. It identifies
what each figure should show, where the data come from, and wording that should
be used or avoided. It does not add scientific functionality and does not claim
finite-temperature MD, MD-ready structures, final elastic constants, or
production mechanical properties.

## Figure 1. Overall Workflow Of pyCSH-Zn

Purpose:

Show the full static construction and validation workflow:

```text
pyCSH brick-based C-S-H generation
-> Zn motif insertion
-> CementFF4-Zn typing and data output
-> static validation
-> LAMMPS read/run0/static minimization
-> post-min validation
-> ensemble analysis
-> selected quasi-static mechanics
```

Data/source:

- `FINAL_WORKFLOW.md`
- `examples/01_generate_pure_csh.py`
- `examples/02_generate_q2b_zn.py`
- `examples/08_generate_q1_zn.py`
- `examples/12_generate_zn_csh_ensemble.py`
- `examples/17_generate_multi_zn_ensemble.py`
- `examples/18_run_selected_multi_zn_mechanics.py`

Suggested caption:

"Schematic of the pyCSH-Zn static candidate workflow, from pyCSH brick-based
C-S-H generation through Zn motif construction, CementFF4-Zn-compatible output,
validation, static minimization, ensemble screening, and selected quasi-static
mechanics."

Do not say:

- MD-ready
- production mechanics
- final elastic constants

## Figure 2. Q1_Zn And Q2b_Zn Motif Definition

Purpose:

Compare the two single-Zn motif classes:

- Q1_Zn: terminal silicate site substitution, corresponding to a
  Q(1,Zn)-related local environment.
- Q2b_Zn: bridging silicate site substitution, corresponding to a
  Q(2p,Zn)-related local environment.

The figure can use a schematic local silicate environment, not a claim of a
unique experimentally proven structure.

Data/source:

- `MANUSCRIPT_METHODS_DRAFT.md`
- `examples/08_generate_q1_zn.py`
- `examples/02_generate_q2b_zn.py`
- `zinc_summary.json` files generated by the workflows.

Suggested caption:

"Motif-level construction of Q1_Zn and Q2b_Zn static candidates. The local
ZnO2(OH)2-style environments are charge-balanced construction motifs used for
screening and diagnostics."

Do not say:

- unique real structure
- experimentally proven Zn environment
- old `mixed_Q1_Q2b_Zn` site type

## Figure 3. Validation And Coordination-Quality Decision Tree

Purpose:

Show the validation decision logic for Zn-O coordination at the unchanged
2.5 Angstrom gate:

```text
any Zn center coordination < 4 -> undercoordinated_failed / failed_multi_zn_candidate
all Zn centers coordination == 4 -> ideal_fourfold
all Zn centers coordination >= 4 and any center > 4 -> overcoordinated
```

Data/source:

- `validate_cementff_data.py`
- `FINAL_WORKFLOW.md`
- `examples/16_screen_multi_zn_combinations.py`
- `examples/17_generate_multi_zn_ensemble.py`

Suggested caption:

"Validation labels and coordination-quality classes used to screen Zn-modified
C-S-H candidates. Overcoordinated candidates are minimum-valid but are not
ideal ZnO4 tetrahedral structures."

Do not say:

- overcoordinated equals ideal ZnO4
- threshold was adjusted

## Figure 5. Quasi-Static Mechanics Protocol

Purpose:

Show the x-direction quasi-static strain protocol:

```text
strain = -0.003, -0.002, -0.001, +0.001, +0.002, +0.003
each strain case starts independently from the same post-min reference
change_box x scale
static minimization
CS-Info reattachment
validation
stress_xx / pressure / energy summaries
```

Data/source:
