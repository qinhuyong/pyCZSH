# v1.7 Multi-Zn Ensemble Generator Status

## Scope

v1.7 adds a constrained multi-Zn ensemble generator:

```text
examples/17_generate_multi_zn_ensemble.py
```

The workflow builds on v1.6-beta single-structure multi-Zn site-combination
screening. Each ensemble model starts from an independent pure C-S-H parent
structure and screens multiple site combinations before selecting a post-min
valid candidate when available.

Supported ensemble modes:

- `multi_q2b_ensemble`: each accepted structure contains multiple Q2b_Zn motifs.
- `multi_q1_ensemble`: each accepted structure contains multiple Q1_Zn motifs.
- `q1_q2b_single_structure_mixed_ensemble`: each accepted structure contains
  Q1_Zn and Q2b_Zn motifs in the same C-S-H structure.
- `mixed_multi_zn_ensemble`: the ensemble contains the three multi-Zn model
  classes above.

## Validation And Selection

The post-min valid criterion is unchanged from v1.6-beta: every Zn center must
have coordination >= 4 within the unchanged 2.5 Angstrom Zn-O validation gate.
If any center has coordination < 4, the model is rejected as a failed multi-Zn
candidate.

Coordination quality is reported separately:

- `ideal_fourfold`: all Zn centers have coordination exactly 4.
- `overcoordinated`: at least one Zn center has coordination greater than 4 and
  no center is undercoordinated.
- `undercoordinated_failed`: at least one Zn center has coordination less than 4.
- `minimum_valid`: all centers satisfy coordination >= 4 when no more specific
  quality label applies.

Candidate ranking prefers post-min valid structures with the matching
`valid_multi_*` label, then `ideal_fourfold`, fewer overcoordinated centers,
lower excess coordination, lower maximum Zn-O distance, and valid Zn-Zn
separation.

## Outputs

Outputs are written under:

```text
output_Y/workflow_v1/multi_zn_ensemble/
```

Top-level outputs include:

- `multi_zn_ensemble_manifest.json`
- `multi_zn_ensemble_summary.csv`
- `multi_zn_ensemble_summary.json`
- `accepted_multi_zn_models.csv`
- `rejected_multi_zn_models.csv`
- `representative_multi_zn_models.json`
- `mechanics_ready_multi_zn_models.csv`
- `motif_survival_summary.csv`
- `coordination_quality_summary.csv`
- `failure_reason_summary.csv`
- optional `plots/*.svg`

Each model directory records input configuration, generation manifest, selected
candidate summary, validation JSON, multi-Zn diagnostics, and post-min data when
static relaxation is requested and succeeds.

## Non-Scope

v1.7 is multi-Zn ensemble generation and analysis only. It is not batch
mechanics, finite-temperature MD, production mechanical-property calculation,
or final elastic constants. `mechanics_ready_multi_zn_models.csv` is a
downstream input manifest only.

This stage does not modify CementFF4-Zn parameters, does not relax the 2.5
Angstrom Zn-O threshold, does not change single-Zn or multi-Zn validation
semantics, does not introduce `md_ready_candidate`, does not enable the old
`mixed_Q1_Q2b_Zn` site type, and does not add Q1_Zn to the default
`examples/07_run_quasistatic_mechanics.py` targets.

Overcoordinated candidates are minimum-valid candidates, not ideal ZnO4
tetrahedral structures. They may be retained and clearly labeled when no
ideal-fourfold representative is available for a class.
