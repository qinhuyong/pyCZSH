# v1.4 Zn-C-S-H Ensemble Generator Status

## Scope

v1.4 adds a constrained random Zn-C-S-H static candidate ensemble generator:

```text
python examples/12_generate_zn_csh_ensemble.py
```

The generator builds on the already validated single-model workflows for:

- pure C-S-H;
- Q2b_Zn static candidates;
- Q1_Zn static candidates.

The output root is:

```text
output_Y/workflow_v1/zn_csh_ensemble/
```

## Supported Ensemble Modes

Supported modes are:

- `q2b_only`
- `q1_only`
- `q1_q2b_mixture`

`q1_q2b_mixture` is an ensemble-level mixture. Each independently generated model is assigned either a Q1_Zn or Q2b_Zn motif according to `--q1-fraction`.

v1.4 does not enable the old `mixed_Q1_Q2b_Zn` site type and does not create a single local mixed Q1+Q2b motif.

## Current Multi-Zn Policy

v1.4 supports one Zn motif per generated structure. The requested `--target-zn-si` is recorded in the manifest and summary, while the actual Zn/Si ratio is reported from each generated model.

Multiple Zn motifs within the same structure are not yet supported in v1.4. This avoids unsafe shared O/H topology, overly short Zn-Zn contacts, and ambiguous charge-balance coupling.

## Constraints

Randomness is constrained by:

- Q1/Q2b topology filters;
- explicit CementFF4-Zn charge assignment;
- charge balance;
- Zn-O coordination validation;
- CS-Info validation;
- water topology validation;
- optional LAMMPS read_check/run0/minimize_static;
- post-min validation when `--run-static-relaxation` is used.

Failed generation, failed initial validation, and failed post-min validation are recorded in `rejected_models.csv`; they do not stop the whole ensemble.

Post-min valid models are accepted only when classified as:

- `valid_q1_zn_candidate`
- `valid_q2b_zn_candidate`

## Non-Scope

This is static candidate ensemble generation. It is not finite-temperature MD, not production mechanical-property calculation, and not final elastic constants.

Mechanics remains a downstream opt-in workflow for post-min valid candidates only.

This stage does not modify CementFF4-Zn force-field parameters, does not relax the 2.5 Angstrom Zn-O validation threshold, does not change validation semantics, does not introduce `md_ready_candidate`, and does not add Q1_Zn to the default `examples/07_run_quasistatic_mechanics.py` targets.

## Example Commands

```text
python examples/12_generate_zn_csh_ensemble.py --n-models 10 --seed-start 1400 --mode q2b_only --target-zn-si 0.05 --run-static-relaxation
python examples/12_generate_zn_csh_ensemble.py --n-models 10 --seed-start 1500 --mode q1_only --target-zn-si 0.05 --run-static-relaxation
python examples/12_generate_zn_csh_ensemble.py --n-models 10 --seed-start 1600 --mode q1_q2b_mixture --q1-fraction 0.5 --target-zn-si 0.05 --run-static-relaxation
```
