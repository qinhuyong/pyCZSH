# v1.5 Ensemble Analysis And Selection Status

## Scope

v1.5 analyzes the ensemble outputs produced by v1.4:

```text
output_Y/workflow_v1/zn_csh_ensemble/
```

The new analysis script is:

```text
examples/13_analyze_zn_csh_ensemble.py
```

This stage only performs statistics, failure reason summaries, mechanics-ready filtering, and representative model selection.

## Outputs

The default output directory is:

```text
output_Y/workflow_v1/zn_csh_ensemble_analysis/
```

Main outputs:

- `ensemble_analysis_summary.json`
- `ensemble_analysis_summary.csv`
- `motif_survival_summary.csv`
- `failure_reason_summary.csv`
- `mechanics_ready_models.csv`
- `representative_models.json`
- `analysis_manifest.json`
- `plots/*.svg` when `--write-plots` is used

## Mechanics-Ready Meaning

`mechanics_ready_models.csv` is only an input manifest for a later v1.6 batch quasi-static mechanics workflow. It does not mean production mechanical-property calculation has been completed.

Representative models are selected from accepted post-minimized candidates using:

- valid Q1_Zn or Q2b_Zn post-min classification;
- Zn-O coordination threshold;
- maximum Zn-O distance threshold;
- Zn-O distance ranking;
- charge residual ranking;
- optional Q1/Q2b balance preference.

## Non-Scope

v1.5 does not support finite-temperature MD, final elastic constants, production mechanical properties, single-structure mixed Q1+Q2b motifs, or single-structure multi-Zn models.

v1.5 does not change CementFF4-Zn force-field parameters, validation semantics, the 2.5 Angstrom Zn-O coordination gate, or the default `examples/07_run_quasistatic_mechanics.py` targets.
