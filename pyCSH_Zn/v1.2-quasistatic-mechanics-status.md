## Strain Cases

For both pure C-S-H and Q2b_Zn:

```text
x_minus_0.003
x_minus_0.002
x_minus_0.001
x_plus_0.001
x_plus_0.002
x_plus_0.003
```

Positive x strain uses:

```text
variable strain equal <magnitude>
variable sx equal 1.0+${strain}
change_box all x scale ${sx} remap
```

Negative x strain uses:

```text
variable strain equal <magnitude>
variable sx equal 1.0-${strain}
change_box all x scale ${sx} remap
```

Each case runs:

```text
run 0
change_box
run 0
minimize
run 0
write_data
reattach CS-Info
validate
```

The mechanics summaries report both the prescribed `strain` and `actual_strain`,
where `actual_strain = final_lx / initial_lx - 1`. Pressure and x-direction
stress are reported in both bar and GPa as `pressure_bar`, `pressure_GPa`,
`stress_xx_bar`, and `stress_xx_GPa`.

## Validation Criteria

A strain case passes if:

- LAMMPS run completes.
- Post-strain data validates as `valid_static_candidate`,
  `valid_q2b_zn_candidate`, or `needs_static_relaxation` where allowed.
- No charge assignment failure.
- No CS-Info failure.
- No bad core-shell pair.
- No severe Zn coordination collapse.
- No severe water topology failure.

must not be reported as final elastic constants or final mechanical-property
results without a later, dedicated analysis and convergence protocol.

No finite-temperature MD is generated or run in v1.2.
