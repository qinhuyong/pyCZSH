# v1.3.4 Q1 Opt-In Mechanics Status

## Scope

v1.3.4 adds an opt-in Q1_Zn quasi-static mechanics workflow. The new script is:

- `examples/11_run_q1_quasistatic_mechanics.py`

The workflow writes its outputs under:

- `output_Y/workflow_v1/mechanics_q1_zn/`

## Q1 Reference Requirement

Q1 mechanics must start from a post-minimized Q1_Zn reference structure that has already validated as:

- `valid_q1_zn_candidate`

If Q1 post-min validation does not pass, the script must not start mechanics from the initial Q1_Zn structure. It must skip or fail clearly instead of silently continuing.

## Strain Protocol

The opt-in Q1 workflow keeps the same six x-direction small-strain cases:

- `-0.003`
- `-0.002`
- `-0.001`
- `+0.001`
- `+0.002`
- `+0.003`

Each strain case starts independently from the same Q1 post-minimized reference. The workflow does not use sequential strain accumulation.

## Default Mechanics Workflow

The default `examples/07_run_quasistatic_mechanics.py` workflow still includes only:

- `pure_csh`
- `q2b_zn`

Q1_Zn mechanics is not added to the default v1.2 mechanics runner.

## Non-Scope

This stage is not finite-temperature MD. It is not a production mechanical-property calculation. It is not a final elastic constants workflow.

This stage does not modify CementFF4-Zn parameters, does not relax the 2.5 Angstrom Zn-O validation threshold, does not change validation semantics, and does not implement mixed Q1+Q2b.
