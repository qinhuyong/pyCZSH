## Implementation Notes

`examples/04_build_lammps_inputs.py` now generates LAMMPS inputs for both:

- `output_Y/workflow_v1/pure_csh/lammps_inputs/`
- `output_Y/workflow_v1/q2b_zn/lammps_inputs/`

Generated static inputs:

- `in.read_check`
- `in.run0`
- `in.minimize_static`
- `in.static_relax_shell`
- `in.elastic_x_plus`
- `in.elastic_x_minus`

`examples/06_run_static_relaxation.py` runs LAMMPS from each input directory so
relative paths resolve correctly.

LAMMPS `write_data` does not preserve the custom `CS-Info` section. The runner
reattaches the original `CS-Info` by atom ID before post-minimization and
elastic smoke-test validation. The validator itself is unchanged.

The x-direction strain templates use safe scale factors:

```text
variable sx equal 1.0+${strain}
variable sx equal 1.0-${strain}
```

These are quasi-static elastic input-validation smoke tests only. This report
does not claim full elastic constants or final mechanical-property results.
