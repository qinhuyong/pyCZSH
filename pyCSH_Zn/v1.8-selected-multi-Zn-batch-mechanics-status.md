# v1.8 Selected Multi-Zn Batch Mechanics Status

## Scope

v1.8 adds an opt-in selected multi-Zn quasi-static mechanics runner:

```text
examples/18_run_selected_multi_zn_mechanics.py
```

The runner reads `mechanics_ready_multi_zn_models.csv` from a v1.7 multi-Zn
ensemble and runs x-direction quasi-static strain cases for selected
post-min-valid multi-Zn models.

This is batch mechanics pipeline validation for selected models. It is not
finite-temperature MD, not final elastic constants, and not a production
mechanical-property workflow.

## Input Gates

The workflow only processes rows that satisfy all of the following:

- `postmin_valid = true`.
- `postmin_validation_label` is one of `valid_multi_q1_zn_candidate`,
  `valid_multi_q2b_zn_candidate`, or `valid_multi_q1_q2b_zn_candidate`.
- `coordination_quality` is not `undercoordinated_failed`.
- `validation_postmin.json` still reports all Zn centers with coordination >= 4.
- `model_dir`, post-min data, and `multi_zinc_summary.json` exist.

`failed_multi_zn_candidate`, `undercoordinated_failed`, and
`postmin_valid = false` models are excluded from mechanics.

If `--require-ideal-fourfold` is used, only `ideal_fourfold` references are run.
If `--include-overcoordinated` is used, overcoordinated minimum-valid references
may run, but their `coordination_quality_reference` is preserved in every
summary.

## Mechanics Policy

The default strain cases are:

```text
-0.003 -0.002 -0.001 +0.001 +0.002 +0.003
```

Every strain case starts independently from the same post-min reference
structure for that model. Sequential strain accumulation is not used. Each
strained structure is statically minimized, reattached with CS-Info, validated,
and summarized with per-center Zn coordination.

## Outputs

Outputs are written under:

```text
output_Y/workflow_v1/selected_multi_zn_mechanics/
```

Top-level outputs include:

- `batch_multi_zn_mechanics_manifest.json`
- `batch_multi_zn_mechanics_summary.csv`
- `batch_multi_zn_mechanics_summary.json`
- `accepted_multi_zn_mechanics_models.csv`
- `failed_multi_zn_mechanics_models.csv`
- `stress_strain_summary_by_model.csv`
- `stress_strain_summary_by_mode.csv`
- `mechanics_failure_reason_summary.csv`
- optional `plots/*.svg`

Each model directory contains `model_mechanics_summary.csv`,
`model_mechanics_summary.json`, `stress_strain.svg`, and one directory per
strain case.

## Non-Scope

v1.8 does not modify CementFF4-Zn force-field parameters, does not relax the
2.5 Angstrom Zn-O coordination threshold, does not change single-Zn or multi-Zn
validation semantics, does not enable the old `mixed_Q1_Q2b_Zn` site type, does
not introduce `md_ready_candidate`, and does not add Q1_Zn to the default
`examples/07_run_quasistatic_mechanics.py` targets.

Overcoordinated models remain minimum-valid mechanics candidates, not ideal ZnO4
tetrahedral structures.
