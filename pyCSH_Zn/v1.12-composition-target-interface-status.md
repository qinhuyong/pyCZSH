# pyCSH-Zn v1.12 Composition-Target Interface Status

v1.12 adds a composition-targeted generation and screening interface for
Zn-modified C-S-H models. The feature targets requested Ca/Si and Zn/Si windows,
records actual final compositions, and keeps the original default workflow
unchanged.

## New Script

`examples/19_generate_composition_targeted_zn_csh.py`

Supported site modes:

- `q2b_only`
- `q1_only`
- `multi_q2b`
- `multi_q1`
- `q1_q2b_single_structure_mixture`

Example:

```bash
python pyCSH_Zn/examples/19_generate_composition_targeted_zn_csh.py --target-ca-si 1.5 --ca-si-tol 0.08 --target-zn-si 0.05 --zn-si-tol 0.03 --site-mode multi_q2b --n-models 5 --seed-start 12000 --output-dir output_composition_targeted
```

## New Shared Parameters

`examples/workflow_common.py::generate_structure()` now accepts optional
composition parameters:

- `target_ca_si`, default `1.7`
- `target_w_si`, default `0.2`
- `ca_si_width`
- `composition_widths`
- `target_zn_si`
- `target_zn_count`

`target_zn_si` and `target_zn_count` are mutually exclusive. For multi-Zn
composition-targeted workflows, `target_zn_si` is converted to
`max(1, round(target_zn_si * N_Si_parent))`. For single-Zn workflows,
`target_zn_si` is recorded as requested metadata and the generator does not
claim to place multiple Zn centers.

## Composition Summary Fields

Composition summaries are written as JSON and CSV and include:

- `N_Ca`
- `N_Si_parent_before_zn`
- `N_Si_final`
- `N_Zn`
- `target_Ca_Si`
- `actual_Ca_Si_final`
- `target_Zn_Si`
- `actual_Zn_Si_final`
- `target_Zn_count`
- `actual_Zn_count`
- `site_mode`
- `seed`
- `within_Ca_Si_tolerance`
- `within_Zn_Si_tolerance`
- `Ca_Si_error`
- `Zn_Si_error`

Ca, Si, and Zn counts use real LAMMPS atom labels/types and do not use oxygen
core-shell counts.

## Target Versus Actual Composition

The inputs are target-window screening values. They are not guaranteed exact
final compositions. Every accepted or rejected model must report
`actual_Ca_Si_final` and `actual_Zn_Si_final`.

Zn/Si control is constrained by integer Zn count, available Q1/Q2b sites,
Zn-Zn compatibility, site screening, and validation.

## Independent Gates

Validation and composition matching are separate gates:

- `validation_passed` comes from the existing validator labels.
- `composition_match` requires both Ca/Si and Zn/Si errors to fall inside their
  requested tolerances.
- `accepted_for_composition_target` requires both gates to pass.

Valid models outside the requested composition window are retained in
`composition_unmatched_valid_models.csv`.

## Frozen Boundaries

- CementFF4-Zn force-field parameters are unchanged.
- The 2.5 Angstrom Zn-O validation threshold is unchanged.
- Existing validation semantics are unchanged.
- The old `mixed_Q1_Q2b_Zn` site type is not enabled.
- No finite-temperature MD is introduced.
- No `md_ready_candidate` is introduced.
- No final elastic constants are claimed.
- No production mechanical properties are claimed.
- `examples/07_run_quasistatic_mechanics.py` default actual targets remain
  `pure_csh` and `q2b_zn`.
- Q1 mechanics and multi-Zn mechanics remain opt-in workflows.
