# Quick Start

Run commands from this package root.

```bash
python pyCSH_Zn/examples/01_generate_pure_csh.py
python pyCSH_Zn/examples/02_generate_q2b_zn.py
python pyCSH_Zn/examples/08_generate_q1_zn.py
python pyCSH_Zn/examples/04_build_lammps_inputs.py
python pyCSH_Zn/examples/06_run_static_relaxation.py
python pyCSH_Zn/examples/07_run_quasistatic_mechanics.py
```

`examples/07_run_quasistatic_mechanics.py` defaults to only:

- `pure_csh`
- `q2b_zn`

In a clean package checkout, `07_run_quasistatic_mechanics.py` requires the
post-minimized pure C-S-H and Q2b_Zn references generated by
`04_build_lammps_inputs.py` and `06_run_static_relaxation.py`.

Q1_Zn mechanics is opt-in:

```bash
python pyCSH_Zn/examples/11_run_q1_quasistatic_mechanics.py
```

Multi-Zn mechanics is opt-in:

```bash
python pyCSH_Zn/examples/18_run_selected_multi_zn_mechanics.py --models-csv pyCSH_Zn/output_Y/workflow_v1/multi_zn_ensemble/mechanics_ready_multi_zn_models.csv --max-models 4 --include-overcoordinated --write-plots
```

For all quasi-static mechanics workflows, strain cases start independently from
the same post-minimized reference structure. Sequential strain accumulation is
not used.
